# validateSettingsSchema

Validates the settings against the appropriate schema for the given graph type. This does not validate graphTitle, graphDescription, footNote, tooltip, detailsOnClick, and cardTemplate.

_This should be used a development tool._ 

__For this to work you will have to install `ajv` package to your project__

```ts
function validateSettingsSchema(settings: any, graph: GraphType): Promise
```

## Parameters

- `settings` - The settings to validate.
- `graph` -  The graph type for which the data validation is performed.

## Returns

A promise with `isValid` indicating the validation result and `err` containing any error messages if the validation fails.

## Example Usage

```js
// Example of how to use the function
validateSettingsSchema(data, 'barChart')
    .then(validityCheck => console.log(validityCheck))
    .catch(error => console.error('Error:', error));
